# 新聞稿 / 商務信件產生器

協助使用者快速產出遊戲公司新聞稿發送 email（主要用途）與一般中文商務信件（次要用途）。核心是「先取得本次要填的變數，再套固定模板產出」。

## 設定（固定資訊）

新聞稿會用到一些每次都一樣的固定資訊：發稿單位、寄件人、媒體聯絡清單、固定結尾語、標題外框。

**重要：claude.ai 的不同對話之間不會自動記住這些設定。** 取得方式依序嘗試：

1. **優先**：若使用者把這些設定寫在 Project（專案）的自訂指令／知識中，直接讀用，不要再問。建議使用者這樣做，一勞永逸。
2. 若使用者在本次對話或上傳的 `config.json` 中提供了，就用那份。
3. 都沒有 → 詢問下列欄位（一次問完）：
   - 發稿單位名稱（如：宇峻奧汀）
   - 寄件人姓名與職稱、聯絡方式
   - 媒體聯絡清單（多筆，名稱＋email）
   - 固定結尾語（預設：敬請您協助發稿，非常感謝。）
   - 新聞稿標題外框（預設：【○○新聞稿】）

問完後，提醒使用者：「若想之後每次對話都不必重填，可把這些設定貼到此 Project 的自訂指令裡。」設定欄位的完整格式見 [references/settings.md](references/settings.md)。

## 每次發稿流程

**每次都先問本次發稿變數，問完才生成標題與內容。** 不要在資訊不齊時就先擬稿。

### 1. 詢問本次變數
- 發布日期（如 2026/05/20，週幾會自動算）
- 遊戲名稱、版號（如 S00，可留空）、改版/活動名稱（可留空）
- 動作（如 今日開服 / 改版上線 / 預約開跑）
- 主標語
- 附件連結
- 收件對象（預設用設定的媒體清單）

缺哪些就問哪些，一次問完。可選欄位沒提供就留空，不要硬編。

### 2. 生成信件
把本次變數與設定分別寫成兩個 JSON 暫存檔，用內附腳本套模板，確保格式（含日期週幾、標點、外框）一致：

```bash
python scripts/render_press_release.py --vars vars.json --config config.json
```

（`--config` 可省略，省略時用腳本內預設值。）腳本會輸出三段：**主旨 / 內文 / 收件人**。模板格式說明見 [references/press-release-template.md](references/press-release-template.md)。

### 3. 呈現給使用者
把主旨、內文放在程式碼區塊內，方便整段複製貼上；收件清單另列。需要使用者補充的地方（如連結還沒給）用備註標在信件外，不要混進正文。

## 關於「定時自動發信」

claude.ai 無法在指定時間自動喚醒並產出信件（這是 Claude Code 桌面工具才有的排程能力）。因此：

- 可以幫使用者整理一份「待發清單」文字（哪天發哪篇），但**不會自動觸發**。
- 屆時請使用者回到對話說「產出待發清單裡的 XXX」，即可立刻生成。
- 若使用者需要真正的定時自動化，建議改用 Claude Code 桌面版的同名外掛。

## 一般商務信件（次要用途）

若要的不是新聞稿，而是詢價、合作提案、跟進、道歉、感謝、通知等一般商務信，改用 [references/templates.md](references/templates.md) 的模板，沿用「先釐清目的與收件人、再套模板」原則。

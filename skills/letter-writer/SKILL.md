# 新聞稿 / 商務信件產生器

協助使用者快速產出遊戲公司新聞稿發送 email（主要用途），以及一般中文商務信件（次要用途）。

核心設計是把資料分三層：**固定設定**（第一次問、存本機、重複用）、**每次發稿變數**（每次都問）、**排程**（指定時間自動產出）。這樣使用者每次只需提供少量本次資訊，其餘自動帶入。

## 設定檔位置

所有設定與排程存在使用者家目錄下：

- 設定檔：`~/.claude/letter-writer/config.json`
- 待發清單：`~/.claude/letter-writer/schedule.json`

（Windows 為 `C:\Users\<使用者>\.claude\letter-writer\`。在 Cowork 等沒有持久家目錄的環境，改存到工作區/outputs 目錄，並告知使用者路徑。）

開始任何工作前，先讀取 `config.json`。讀不到代表是第一次使用，進入「第一次設定」。

## 第一次設定（找不到 config.json 時）

逐項詢問以下設定並存成 `config.json`。可一次列出問題請使用者一起回答，避免逐條來回。完整欄位與格式見 [references/config-schema.md](references/config-schema.md)。

1. 發稿單位名稱（預設：宇峻奧汀）
2. 寄件人姓名與職稱
3. 寄件人聯絡方式（email / 電話）
4. 媒體聯絡清單（多筆，每筆含名稱與 email）
5. 固定結尾語（預設：敬請您協助發稿，非常感謝。）
6. 常用遊戲產品清單（如《遊戲名稱》）
7. 新聞稿標題外框（預設：【宇峻奧汀新聞稿】）

存檔後告知使用者「設定已儲存，之後可直接發稿」，再接著進行本次發稿。

## 每次發稿流程

**重要：每次都要先問本次發稿變數，問完才開始生成標題與內容。** 不要在資訊不齊時就先擬稿。

### 1. 詢問本次變數
即使有設定檔，每次發稿仍須向使用者取得這些「會變」的欄位：

- **發布日期**（如 2026/05/20，週幾會自動算）
- **遊戲名稱**（可從常用清單挑，或新輸入）
- **版號**（如 S00，可留空）
- **改版/活動名稱**（如 改版名稱，可留空）
- **動作**（如 今日開服 / 改版上線 / 預約開跑）
- **主標語**（如 主標語○○○）
- **附件 Google Drive 連結**
- **收件對象**（預設用設定檔的媒體清單，可調整）
- **預定寄送時間**（可選；有給就進排程）

缺哪些就問哪些，一次問完。使用者沒提供版號或改版名等可選欄位時，留空即可，不要硬編。

### 2. 生成信件
把變數寫成 JSON，用內附腳本套模板產出，確保格式（含日期週幾、標點、外框）每次一致：

```bash
python skills/letter-writer/scripts/render_press_release.py \
  --vars <本次變數.json> --config ~/.claude/letter-writer/config.json
```

在 Windows 上執行時，請設環境變數避免中文輸出亂碼：`PYTHONUTF8=1`（PowerShell：`$env:PYTHONUTF8=1`）。

腳本會輸出三段：**主旨 / 內文 / 收件人**。模板格式與佔位欄位說明見 [references/press-release-template.md](references/press-release-template.md)。

### 3. 呈現給使用者
把主旨、內文放在程式碼區塊或清楚分隔線內，方便整段複製貼上。收件清單另列。若有需要使用者補充的地方（如連結還沒給），用備註標在信件外，不要混進正文。

## 排程（指定時間自動產出）

當使用者給了「預定寄送時間」，做兩件事：

### A. 登記待發清單（一定要做）
把本次發稿的變數與時間寫入 `schedule.json` 的 `pending` 陣列（格式見 [references/scheduling.md](references/scheduling.md)）。這份清單可隨時查詢「有哪些稿待發、何時發」，即使排程功能不可用也保留得住。

### B. 建立真實排程（環境支援時）
若環境支援排程，建立一個排程任務，在指定時間喚醒 Claude、讀取該筆 `schedule.json` 紀錄、用腳本生成信件並呈現/寄出。具體做法（用 `/schedule` 或 scheduled-tasks）見 [references/scheduling.md](references/scheduling.md)。若環境不支援，明確告訴使用者「已登記待發清單，但此環境無法自動觸發，屆時請手動執行」。

## 一般商務信件（次要用途）

若使用者要的不是新聞稿，而是詢價、合作提案、跟進、道歉、感謝、通知等一般商務信，改用 [references/templates.md](references/templates.md) 的模板，沿用同樣的「先釐清目的與收件人、再套模板」原則。設定檔中的寄件人資訊與署名仍可沿用。

## Cowork 注意事項

在 Cowork 環境：沒有瀏覽器與持久家目錄，設定檔/清單改存工作區或 outputs 目錄並回報路徑；產出的信件直接貼在對話中，需要檔案時存到 outputs 供下載。
